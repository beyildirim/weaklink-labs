For the action plan to reach SLSA Level 3, focus on these specific CI/CD changes:

1. **Provenance generation**: Use `slsa-github-generator` or `slsa-verifier`
   to produce SLSA provenance as a build step. This is the single biggest
   step from Level 0/1 to Level 2.

2. **Build isolation**: Ensure each build runs in a fresh, ephemeral
   container or VM. No shared state between builds. GitHub Actions runners
   already provide this if you use GitHub-hosted runners.

3. **Unforgeable provenance**: The provenance must be generated by the
   build platform, not by your build script. With `slsa-github-generator`,
   the provenance is generated by a reusable workflow that your build
   cannot tamper with.

4. **Source verification**: The build service must verify where the source
   came from. This is handled automatically by GitHub Actions when using
   the SLSA generator.

For each gap, specify: what tool or service, what configuration change,
who owns it, and a realistic timeline.
